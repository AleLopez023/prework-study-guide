Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git clone: Git clone is a command for downloading existing source code from a remote repository (like Github, for example). In other words, Git clone basically makes an identical copy of the latest version of a project in a repository and saves it to your computer. This will make a copy of the project to your local workspace so you can start working with it.
  • git branch: Creates a new branch locally (git branch "branch-name").
  • git push: Pushes the branch into the remote repo. (git push -u "remote" "branch-name")
  • git branch or git branch --list: Allows you to view branches.
  • git branch -d "branch-name": Allows you to delete a branch.
  • git add: Adds a single file. You can also use git add -A to add everthing at once. The git add command doesn't change the repository and the changes are not saved until we use git commit.
  • git commit: Is like setting a checkpoint in the development process which you can go back to later if needed. Be sure to add a short message to explain what was developed or change with git commit -m "commit message". IMPORTANT: git commit saves your changes only locally.
  • git push: Uploads your commits to the remote repository. The main way to do this is git push "remote" "branch-name". If a new branch was created, then you have to upload the branch with the following command: git push --set-upstream "remote" "name-of-your-branch"or git push -u origin "branch_name". IMPORTANT: git push only uploads changes that are committed.
  • git pull: Used to get updates from the remote repo. This command is a combination of git fetch and git merge which means that, when we use git pull, it gets the updates from remote repository (git fetch) and immediately applies the latest changes in your local (git merge). git pull "remote"
  • git merge: When you're finished developing, allows you to merge the branch to the main branch. It's important to remember that you first need to be on the specific branch that you want to merge with your feature branch. An example would be as follows:
    1. Switch to main branch: git checkout main
    2. Before merging, update you local main branch: git fetch
    3. Merge your feature branch into the main branch: git merge "branch-name"
