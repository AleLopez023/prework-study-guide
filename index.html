Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git clone: Git clone is a command for downloading existing source code from a remote repository (like Github, for example). In other words, Git clone basically makes an identical copy of the latest version of a project in a repository and saves it to your computer. This will make a copy of the project to your local workspace so you can start working with it.
  • git branch: Creates a new branch locally (git branch "branch-name").
  • git push: Pushes the branch into the remote repo. (git push -u "remote" "branch-name")
  • git branch or git branch --list: Allows you to view branches.
  • git branch -d "branch-name": Allows you to delete a branch.
  • git add: Adds a single file. You can also use git add -A to add everthing at once. The git add command doesn't change the repository and the changes are not saved until we use git commit.
  • git commit: Is like setting a checkpoint in the development process which you can go back to later if needed. Be sure to add a short message to explain what was developed or change with git commit -m "commit message".
    IMPORTANT: git commit saves your changes only locally.
  • git push: Uploads your commits to the remote repository. The main way to do this is git push "remote" "branch-name". If a new branch was created, then you have to upload the branch with the following command: git push --set-upstream "remote" "name-of-your-branch"or git push -u origin "branch_name".
    IMPORTANT: git push only uploads changes that are committed.
  • git pull: Used to get updates from the remote repo. This command is a combination of git fetch and git merge which means that, when we use git pull, it gets the updates from remote repository (git fetch) and immediately applies the latest changes in your local (git merge). git pull "remote"
  • git merge: When you're finished developing, allows you to merge the branch to the main branch. It's important to remember that you first need to be on the specific branch that you want to merge with your feature branch. An example would be as follows:
    1. Switch to main branch: git checkout main
    2. Before merging, update you local main branch: git fetch
    3. Merge your feature branch into the main branch: git merge "branch-name"

JavaScript

  • No one uses "var", so don't use it either.
  • const: keyword that allows you to declare a variable. This variable cannot be reassigned.
  • let: keyword that allows you to reassign the value of a variable.
  • A value is considered falsy if it is one of the following values: 0, -0, On, "", null, undefined, NaN, or a Boolean data type with the value of false.
  • All other values, including a Boolean with the value of true, are considered truthy.
  • When the computer encounters an if statement in a JavaScript file, it will check to see if the condition that we provide is truthy. If it is, then it will do whatever action we ask it to do in between the curly brackets. If the condition isn’t true or has a value that isn’t truthy, then the computer will ignore whatever is in between the curly brackets and move on.
  • A for loop uses the predictable pattern of indices to perform a task on all the items in an array by allowing a single code block to be executed over and over. Developers must use another statement, such as the conditional if statement that we just learned, to interrupt the control flow by executing a block of code a specific number of times
  • A for loop contains the following three important statements:
    1. The first statement determines the starting point for our loop. Since we want to start at the beginning of the array, we declare a variable x and give it a value of 0 (because the first index of an array is always 0), as shown in the following line of code: let x = 0;
    2. The second statement is the condition. As long as the number in the variable x is less than the length of the array, the loop will keep running. This will allow the loop to execute one time for each item in the array. We use the length built-in array property on our shapes array to express that condition, as shown in the following line of code: x < shapes.length;
    3. The final statement is what allows the array to iterate over each item. Since indices follow a predictable pattern, we know the number increases by 1 each time. The code x++ is a shortcut way of writing x+1. Now every time the code loops, x will increase by 1, as shown in the following line of code: x++
    4. Final code would look like:
      for(let x =0; x < shapes.length; x++) {
      // code block ex. console.log(shapes[x]);
      }
